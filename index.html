TaskHive App Showcase

COP 4331 – Object Oriented Software Engineering · Florida Atlantic University · Fall 2025
Author: Adam Wallace · Project: TaskHive – Multi-List Task Manager

TaskHive is a Java desktop application that helps you manage multiple task lists with due dates, priorities, and categories. It was built as a semester project for COP 4331 – Object Oriented Software Engineering at Florida Atlantic University.

Overview

TaskHive was created for students and busy users who need several separate lists instead of one giant to-do list where items get lost. You can maintain different lists for:

  • Personal goals – self-care, workouts, habits
  • Academic work – homework, projects, exams
  • Professional tasks – meetings, deliverables, deadlines

Each list contains tasks with their own due dates and priorities, and tasks are visually organized by urgency.

Java 17 Swing GUI MVC Architecture Local File Persistence

Core Features

Lists

  • Create lists with optional description, category, due date, and priority.
  • Switch between different lists from a central dashboard.
  • Delete entire lists (with confirmation prompts).

Tasks

  • Add tasks to a list with name, description, optional due date, and priority.
  • Edit existing tasks or delete them from the list.
  • Mark tasks complete; lists become complete when all tasks are done.

Due Dates, Priority & Visual Feedback

  • Per-task due dates, plus an optional due date for the entire list (e.g., project deadline).
  • Priority and color coding for urgency (low / normal / high).
  • Visual indicators for overdue, late, and completed tasks.

Search, Filter & Sort

  • Search lists and tasks by keywords.
  • Filter by category, created date, due date, or priority.
  • Sort by due date and urgency to see what’s most important first.

Local Persistence

  • All lists and tasks are stored locally on disk.
  • Data is re-loaded automatically when you reopen the application.

Architecture & Design Patterns

TaskHive was designed with a Model–View–Controller (MVC) structure and several GoF patterns to keep the code modular, testable, and maintainable.

MVC Layers

Model

  • Task – represents a single task.
  • TaskListModel – represents a list containing many tasks.
  • TaskRepository – stores and loads all task lists from disk.

View

  • DashboardView – main window with all task lists.
  • TaskListView – window showing tasks for a single list.
  • TaskRowPanel & TaskListRowPanel – reusable Swing components for rows.

Controller

  • DashboardController
  • TaskListController

Main Patterns Used

Repository Pattern

TaskRepository handles loading and saving all TaskListModel objects and hides file I/O from the rest of the application. Controllers interact with clean repository methods instead of dealing with files directly.

Composite Pattern

TaskListModel behaves as a composite that owns many Task entities. The UI can treat a whole list as one object (add, delete, mark complete) while internally it manages a collection of child tasks.

Strategy Pattern (Filtering & Search)

Different filter strategies (by category, due date, priority, or none) can be swapped at runtime via a context class. This allows the dashboard to change its filtering behavior without modifying controller code.

Factory / Dialog Factory

NewListDialog and NewTaskDialog act as factories that create TaskListModel and Task objects from user input. In the full design, a dedicated DialogFactory can instantiate dialogs so controllers remain focused on coordination logic instead of UI construction.

How to Run TaskHive Locally

  1. Clone the TaskHive source repository from GitHub.
  2. Open the project in an IDE such as IntelliJ IDEA, Eclipse, or NetBeans.
  3. Ensure you are using Java 17+.
  4. Build / compile the project.
  5. Run the main class to launch the TaskHive desktop application.
